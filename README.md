# SQL
Proyecto: DataProject: Lógica. Consultas de SQL

1.	DESCRIPCIÓN DEL PROYECTO

Este proyecto consiste en el análisis y explotación de una base de datos relacional (modelo tipo DVD Rental / Sakila) utilizando SQL y PostgreSQL, trabajando desde la herramienta DBeaver.

El objetivo principal ha sido consultar, analizar y extraer información relevante de la base de datos siguiendo buenas prácticas SQL, resolviendo una serie de enunciados progresivos que abarcan desde consultas básicas hasta consultas avanzadas con subconsultas, funciones de agregación, joins, vistas y tablas temporales.


2.	HERRAMIENTAS UTILIZADAS

- PostgreSQL
- DBeaver
- GitHub


3.	ESTRUCTURA DEL PROYECTO

- Consultas SQL organizadas por enunciados
- Uso de vistas y tablas temporales
- Archivo README.md con la documentación del proyecto


4.	BUENAS PRÁCTICAS APLICADAS

Durante todo el desarrollo del proyecto se han aplicado las siguientes buenas prácticas:
-	Uso de palabras clave SQL en MAYÚSCULAS (SELECT, FROM, WHERE, etc.)
-	Finalización de todas las consultas con ;
-	Uso de comillas dobles para tablas y columnas (PostgreSQL)
-	Evitar el uso de SELECT *, seleccionando solo las columnas necesarias
-	Uso de alias para mejorar la legibilidad en consultas con JOIN
-	Inclusión de comentarios para documentar cada consulta
-	Uso de ROUND(…, 2) en todos los cálculos solicitados (promedios, medias, varianzas…)
-	Separación clara de cada ejercicio mediante comentarios


5.	CONTENIDOS TRABAJADOS

A lo largo del proyecto se han trabajado los siguientes conceptos:

-	Consultas básicas 
-	Funciones de agregación
-	Subconsultas
-	Joins
-	Fechas y tiempo
-	Vistas y tablas temporales


6.	ANÁLISIS REALIZADO: 

Gracias a las consultas desarrolladas, se ha podido:

-	Analizar la distribución de películas por categoría y clasificación
-	Estudiar duraciones máximas, mínimas y medias
-	Identificar actores más activos y nombres más repetidos
-	Analizar el comportamiento de alquileres por día y por mes
-	Detectar clientes más activos y con mayor gasto
-	Relacionar películas, actores, categorías, inventario y alquileres

Este análisis permite obtener una visión global del negocio, tanto a nivel de catálogo como de clientes y alquileres.


7.	CONCLUSIONES

El proyecto ha permitido afianzar conocimientos clave de SQL, especialmente:
-	La importancia de entender bien el modelo relacional
-	El uso correcto de joins según el resultado esperado
-	La diferencia entre filtrar filas (WHERE) y filtrar grupos (HAVING)
-	El valor de documentar y estructurar correctamente el código
